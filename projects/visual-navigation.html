// Project Detail — Hancom InSpace Co., Ltd. · Feb 2021 – Dec 2021

Visual Navigation
& Sensor Fusion

Stereo camera-based visual perception pipeline for terrain-relative navigation — producing rectified imagery, disparity maps, and dense 3D point clouds to support autonomous localization in GPS-denied environments.

Stereo Vision Point Cloud Disparity Sensor Fusion ROS Hancom InSpace · 2021

// 01 · Input — Stereo Camera

Raw Stereo Image Pair

The perception pipeline begins with a calibrated stereo camera capturing synchronized left and right image pairs. This dual-camera setup provides the baseline separation needed for triangulation-based depth estimation, enabling the system to reconstruct 3D structure from 2D imagery without any additional active sensor.

Raw stereo image pair — synchronized left and right camera frames from the stereo rig.

Tech Stack Stereo Camera · ROS image_transport · Camera Calibration (OpenCV) · Synchronized Image Capture

// 02 · Processing — Image Rectification

Stereo Rectification

Raw stereo images are first corrected for lens distortion and stereo misalignment through rectification. After rectification, corresponding points between the left and right images lie on the same horizontal scanline — a necessary prerequisite for efficient stereo matching. The rectified image pair is the input to the disparity computation stage.

Rectified stereo image — lens distortion corrected and epipolar lines aligned horizontally for stereo matching.

// 03 · Processing — Disparity Estimation

Disparity Heatmap

Stereo block matching is applied to the rectified image pair to compute a dense disparity map — measuring the horizontal pixel displacement between corresponding points in the left and right images. Disparity is inversely proportional to depth: high disparity (warm colors) indicates objects close to the camera, while low disparity (cool colors) indicates distant objects.

Disparity heatmap — dense stereo matching result visualized as a color-coded depth map (warm = near, cool = far).

Tech Stack Semi-Global Block Matching (SGBM) · OpenCV · ROS stereo_image_proc · WLS Filter (disparity refinement)

// 04 · Output — 3D Point Cloud

Dense 3D Point Cloud

The disparity map is back-projected into 3D space using the stereo camera's calibrated parameters to produce a dense point cloud. Each pixel in the disparity image becomes a 3D point with known XYZ coordinates relative to the camera frame. This real-time 3D representation of the environment forms the core input for terrain-relative navigation and localization.

Dense 3D point cloud reconstructed from stereo disparity — real-time scene geometry for terrain-relative navigation.

Tech Stack Stereo Reprojection (Q-matrix) · PCL (Point Cloud Library) · ROS PointCloud2 · Real-time 3D Reconstruction

// 05 · Output — Filtered Point Cloud

Point Cloud Filtering & Trimming

The raw dense point cloud contains noise from stereo mismatches and out-of-range measurements. A statistical outlier removal filter is applied, followed by range-based trimming to retain only points within the relevant operational volume. The resulting filtered point cloud provides a cleaner, more reliable 3D map of the immediate environment — directly usable for obstacle avoidance and terrain matching.

Filtered point cloud — outlier-removed and range-trimmed 3D map for reliable terrain-relative localization.

Tech Stack PCL StatisticalOutlierRemoval · PassThrough Filter · VoxelGrid Downsampling · ROS · Python / C++
